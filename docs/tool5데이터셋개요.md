### Tool5 데이터셋

- **Input:** Tool4 결과 (차이점 목록), 원본 청구항, OA 거절이유
- **Output:** 보정된 청구항 텍스트

**정답셋 구성**

KIPRIS API에서 보정서 가져옴
- 삭제(Del) 수정(Ins) 태그가 달린 보정항을 구할 수 있어서 그걸로 쌍으로 가능. 

**학습 전략**

| 단계 | 방법 |
|---|---|
| 1단계 SFT | Input: OA + 원본 청구항 + 차이점 / Output: 보정항 (정답) → 모범답안 보고 공부 |
| 2단계 KTO | 선호 예측 모델이 생성된 보정항 채점 → 등록될 것 같음 (desirable) / 등록 안 될 것 같음 (undesirable) → 그 신호로 추가 정렬 |

또는 단순히 CE

**성능평가지표**

Tool4, Tool5의 결과물은 답이 여러 개라 단순 텍스트 유사도 및 도메인 학습이 안 된 LLM judge로 평가하기 어려울 수 있음.

사람 검수 (최종): 통계적 샘플링 100건 내외 → 실제로 거절이유가 극복됐는지 확인 → 선호 예측 모델 신뢰도 검증 용도

또는 여건이 안된다면 ClaimBrush 논문 등 여러 논문 참고해서 선호 예측 모델을 학습시켜서 성능평가지표로 사용할 수 밖에 없음
