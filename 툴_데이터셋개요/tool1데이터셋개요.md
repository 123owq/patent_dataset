### Tool1 데이터셋
(프롬프팅으로 충분할 수 있음)

- **Input:** 의견제출통지서(OA) 텍스트 (pdf를 코드 기반 파싱한 원문 텍스트)
- **Output:** 심사관 거절이유 구조화 JSON
  - 거절 유형 (신규성/진보성/기재불비 등) 및 특허법 조항
  - 해당 청구항 번호
  - 인용 선행특허 번호
  - 심사관 논리 

**테이터셋 구성** 
tool1 데이터셋은 학습용x 평가용(벤치마크)

- KIPRIS에서 실제 OA 문서 수집 pdf는 스캔본이 아니라 파싱가능
- 정답: OA 본문에 명시된 정보를 코드기반 파싱 후 llm 기반 JSON 형태 변환 후 필요시 사람검수

**성능평가지표**

- 청구항 번호: 숫자 set 비교 (F1)
- 선행특허 번호: 정규식 정규화 후 exact match (F1)
- 거절유형: 분류 정확도 (Accuracy)
- 심사관 논리: LLM judge
- 구조화 완성도: 필수 필드 누락 여부
