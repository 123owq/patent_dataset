### Tool2 데이터셋
(프롬프팅으로 충분할 수 있음)

- **Input:** 특허 청구항 텍스트 (독립항 + 종속항 전체)
- **Output:** 파싱된 청구항 구조체
  - 청구항 번호
  - 독립항 / 종속항 구분
  - 인용 청구항 번호 (종속항의 경우)
  - 구성요소(element) 단위로 분해된 리스트
  - 원문 텍스트 (raw_text)
  - element별 관련 상세설명 구절 매핑

**데이터셋 구성**
tool2 데이터셋은 학습용x 평가용(벤치마크)

- KIPRIS API로 한국타이어 특허 pdf 수집
- 정답: 코드 기반 파싱 후 LLM 처리로 구조체 생성, 필요시 사람 검수
- 유의점: 회사마다 청구항 작성 스타일이 달라 용어·문체 차이 존재

**성능평가지표**

- 청구항 번호: 숫자 set 비교 (F1)
- 독립/종속 구분: 정규식("제N항에 있어서" 패턴) 기반 exact match
- 인용 청구항 번호: 정규식 정규화 후 exact match (F1)
- 구성요소 분해: 싼 모델 예측 → 비싼 모델 LLM judge
- 상세설명 매핑: LLM judge (매핑 적절성)
